Linux / macOS

Bash, zsh, Git Bash — chemins Unix, export, curl

Installation

curl -fsSL https://x.ai/cli/install.sh | bash

# Version spécifique
curl -fsSL https://x.ai/cli/install.sh | bash -s 0.1.42

# Vérifier & mettre à jour
grok --version
grok update
Git Bash (Windows) : utilise aussi ce script bash si tu préfères PowerShell.

Lancement & authentification

grok Lance l'interface TUI interactive
grok "ta tâche" Lance le TUI avec un prompt initial
grok --yolo Auto-approuve toutes les actions
grok -m grok-build Choisir un modèle spécifique
grok --cwd ~/projets/mon-app Démarrer dans un répertoire précis
grok -c Continuer la session la plus récente
grok --resume <id> Reprendre une session par son ID
grok login Connexion via navigateur (OAuth)
grok login --device-auth Connexion sans navigateur (code appareil) 
# Auth par clé API (CI / sans navigateur)
export XAI_API_KEY="xai-..."
grok

Mode headless (scripts & CI)

grok -p "ta tâche" Exécuter une tâche sans interface
grok -p "..." --yolo Headless avec auto-approbation
grok -p "..." --output-format json Sortie JSON (text, sessionId, stopReason)
grok -p "..." --output-format streaming-json Flux NDJSON en temps réel
grok -p "suite" --resume <id> Continuer une session headless
grok --prompt-file ./prompt.txt Lire le prompt depuis un fichier 
# Exemple CI avec jq
grok -p "Review changes" --output-format json --yolo | jq -r '.text'

# Contexte git dans le prompt
grok -p "Commit message pour: $(git diff --staged)"

CLI utilitaires

Windows Shell

PowerShell, Windows Terminal, VS Code / Cursor — chemins Windows, $env:

Installation (PowerShell)

irm https://x.ai/cli/install.ps1 | iex

# Version spécifique
$env:GROK_VERSION="0.1.42"
irm https://x.ai/cli/install.ps1 | iex

# Vérifier & mettre à jour
grok --version
grok update
PATH : le script ajoute %USERPROFILE%\.grok\bin automatiquement. Alternatives : Git Bash (curl | bash) ou WSL.

Lancement & authentification

grok Lance l'interface TUI interactive
grok "ta tâche" Lance le TUI avec un prompt initial
grok --yolo Auto-approuve toutes les actions
grok --cwd C:\Users\toi\projets\mon-app Démarrer dans un répertoire précis
grok -c Continuer la session la plus récente
grok --resume <id> Reprendre une session par son ID
grok login Connexion via navigateur (OAuth)
grok login --device-auth Connexion sans navigateur (code appareil) 
# Auth par clé API (CI / sans navigateur)
$env:XAI_API_KEY="xai-..."
grok

# Persister la variable (session utilisateur)
[Environment]::SetEnvironmentVariable("XAI_API_KEY", "xai-...", "User")

Mode headless (scripts & CI)

grok -p "ta tâche" Exécuter une tâche sans interface
grok -p "..." --yolo Headless avec auto-approbation
grok -p "..." --output-format json Sortie JSON
grok -p "suite" --resume <id> Continuer une session headless
grok --prompt-file .\prompt.txt Lire le prompt depuis un fichier 
# Exemple PowerShell
grok -p "Review changes" --output-format json --yolo | ConvertFrom-Json | Select-Object -ExpandProperty text

Alt+V est spécifique à Grok sur Windows — Ctrl+V ne gère pas les images du presse-papiers. Lance /terminal-setup pour diagnostiquer couleurs et clipboard.
Windows Terminal : pour coller des images avec Ctrl+V, ajoute { "command": null, "keys": "ctrl+v" } dans actions de settings.json.

Slash commands

/new Nouvelle session (alias : /clear)
/resume Ouvrir le sélecteur de sessions
/model grok-build Changer de modèle (alias : /m)
/always-approve Mode YOLO (alias : /yolo)
/compact Compresser l'historique
/context Usage du contexte
/settings Paramètres (alias : /config)
/plugins Plugins & marketplace
/skills Gérer les skills
/mcps Serveurs MCP
/plan Mode planification
/terminal-setup Diagnostic terminal
/home Écran d'accueil (alias : /welcome)
/quit Quitter (alias : /exit)
Tape / pour l'autocomplétion. Référence fichiers : @src/main.rs ou @src/main.rs:10-50
