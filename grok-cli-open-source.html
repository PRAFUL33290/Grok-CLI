Grok CLI Open Source transforme votre terminal en un copilote IA puissant qui comprend votre codebase, maintient le contexte sur plusieurs sessions, et peut déléguer du travail à des sous-agents. Il est conçu pour les développeurs qui veulent garder le contrôle total dans leur environnement CLI tout en bénéficiant de l'IA.

Points forts du projet

🧠

Contexte persistant

Le contexte est maintenu à travers les sessions de terminal. L’agent se souvient des fichiers, décisions architecturales et de l’historique du projet.

🔍

Recherche en temps réel

Accès natif à search_x (X/Twitter) et search_web directement depuis l’agent.

🖼️

Génération multimédia

Génération d’images et de vidéos courtes intégrée (text-to-image, image-to-video, édition). Fichiers sauvegardés localement.

📱

Contrôle à distance

Contrôle via Telegram : pairage unique, puis pilotage complet de l’agent depuis votre téléphone.

Fonctionnalités avancées

Sub-agents par défaut Delegation parallèle (explore, general, computer, verify...)
Computer Use Agent desktop pour automatiser macOS via accessibilité + snapshots sémantiques
Skills Workflows réutilisables dans .agents/skills/ ou ~/.agents/skills/
MCP Servers Extensible via Model Context Protocol (même standard que l’outil officiel)
Verify /verify ou --verify : build, test, capture d’écran et vidéo dans un sandbox
Sandbox (Shuru) Exécution dans une microVM isolée sur macOS Apple Silicon
Hooks Exécution de commandes shell sur les événements du cycle de vie de l’agent
AGENTS.md Fichiers d’instructions projet (style Codex) fusionnés automatiquement

Installation

Installation rapide (recommandée) :

curl -fsSL https://raw.githubusercontent.com/superagent-ai/grok-cli/main/install.sh | bash

Alternative avec Bun :

bun add -g grok-dev
Prérequis : Une clé API Grok valide (obtenue sur x.ai) + un terminal moderne (WezTerm, Alacritty, Ghostty ou Kitty recommandés pour la TUI).

Comment l’utiliser

Mode interactif (TUI) :
grok ou grok -d /chemin/du/projet
Mode headless (scripts / CI) :
grok --prompt "Exécute les tests et résume les échecs" --format json
Contrôle à distance Telegram :
/remote-control → Telegram dans la TUI, puis /pair depuis Telegram.
